# MockInterview 部署方案与盈利分析

> 更新日期: 2026-04-06

## 一、部署方案对比

### 当前架构（云服务）

| 组件 | 服务商 | 说明 |
|------|--------|------|
| 前端 + API | Vercel | Next.js 托管，自动扩容 |
| Python Agent | Railway / Fly.io | LiveKit Agents 语音代理 |
| 数据库 | Neon | Serverless PostgreSQL |
| LLM | DeepSeek API | 兼容 OpenAI 格式，性价比高 |
| STT | Deepgram Nova-3 | 流式语音识别，准确率高 |
| TTS | ElevenLabs | 高质量语音合成（成本最高） |
| 实时通信 | LiveKit Cloud | WebRTC SFU，免费层 5000 min/月 |

### 自建方案

| 组件 | 替代方案 | 说明 |
|------|---------|------|
| 前端 + API | Docker + Nginx | Next.js standalone 模式 |
| Python Agent | Docker | 同一服务器或独立容器 |
| 数据库 | PostgreSQL Docker | 标准 PostgreSQL 实例 |
| LLM | DeepSeek API / Ollama 本地 | API 便宜，本地需 GPU |
| STT | faster-whisper | 本地 GPU 推理，免费 |
| TTS | edge-tts / ChatTTS | 微软免费 API / 本地模型 |
| 实时通信 | LiveKit Server (开源) | Go 项目，Docker 部署 |

### 混合方案（推荐起步）

自建 TTS（edge-tts 免费）+ 自建 STT（faster-whisper）+ 云 LLM（DeepSeek API）+ LiveKit Cloud 免费层

## 三、运营成本详算

### 假设条件

- 日均 100 次面试，每次 30 分钟
- 用户结构：70% 免费 / 25% PRO(¥39) / 5% PREMIUM(¥99)
- 每用户平均收入 (ARPU)：¥14.7/月
- 免费用户月均 2 次面试，PRO 月均 12 次，PREMIUM 月均 20 次
- 加权平均每用户每月 5.4 次面试，约 150 分钟语音

### 方案一：云服务（使用 edge-tts 替代 ElevenLabs）

**固定成本：¥310/月**

| 项目 | 月费 |
|------|------|
| Vercel Pro | ¥150 |
| Neon Pro | ¥140 |
| 域名 + 备案 | ¥20 |

**变动成本：¥9.7/用户/月**

| 服务 | 计算方式 | 单用户月成本 |
|------|---------|------------|
| DeepSeek API (LLM) | 150min × ~2000 token/min × ¥2/百万 | ¥0.6 |
| Deepgram (STT) | 150min × $0.0043/min × 7.3汇率 | ¥4.7 |
| edge-tts (TTS) | 免费 | ¥0 |
| LiveKit Cloud | 150min × $0.004/min × 7.3 | ¥4.4 |
| **合计** | | **¥9.7** |

**毛利：¥14.7 - ¥9.7 = ¥5.0/用户/月**

> 注意：如果使用 ElevenLabs 而非 edge-tts，变动成本飙升到 ¥31/用户/月，直接亏损。edge-tts 是云服务方案盈利的前提。

### 方案二：自建

**固定成本：¥1,420/月**

| 项目 | 月费 |
|------|------|
| 应用服务器 (2核4G 云主机) | ¥100 |
| GPU 服务器 (4090 16G 显存) | ¥1,200 |
| 域名 + 带宽 (~500GB/月) | ¥120 |

**变动成本：¥0.6/用户/月**

| 服务 | 方案 | 单用户月成本 |
|------|------|------------|
| DeepSeek API (LLM) | 云端按量付费 | ¥0.6 |
| faster-whisper (STT) | 本地 GPU 推理 | ¥0 |
| edge-tts (TTS) | 免费 API | ¥0 |
| LiveKit Server | 自建 | ¥0 |
| **合计** | | **¥0.6** |

**毛利：¥14.7 - ¥0.6 = ¥14.1/用户/月**

> GPU 服务器并发上限约 50-80 同时在线用户。超过需加机器，每台 +¥1,200/月。

## 六、推荐路线

### 阶段一：MVP 验证期（0-200 用户）

- **方案：云服务 + edge-tts**
- 月成本：¥310 固定 + ¥9.7/用户
- 目标：62 用户达到盈亏平衡
- 重点：验证产品、获取用户反馈

### 阶段二：增长期（200-1000 用户）

- **方案：逐步迁移到自建**
- 优先自建：TTS (edge-tts, 已完成) → STT (faster-whisper) → LiveKit Server
- LLM 继续用 DeepSeek API（成本极低，无需自建）
- 利润率从 30% 提升到 80%+

### 阶段三：规模化（1000+ 用户）

- **方案：全面自建**
- 多台 GPU 服务器 + 负载均衡
- 考虑自建 LLM（如果 API 成本成为瓶颈）
- 月利润可达 ¥12,000+

## 七、关键风险

| 风险 | 影响 | 应对 |
|------|------|------|
| edge-tts 免费策略变化 | TTS 成本突增 | 备选 ChatTTS 本地部署 |
| DeepSeek API 涨价 | LLM 成本增加 | 可切换到 Ollama 本地模型 |
| GPU 服务器价格波动 | 自建成本不确定 | 签长期合约或购买自有 GPU |
| 用户付费率低于预期 | ARPU 降低 | 调整免费层限制（如 1 次/月） |
| 语音延迟影响体验 | 用户留存下降 | 优先保证 STT/TTS 质量 |
